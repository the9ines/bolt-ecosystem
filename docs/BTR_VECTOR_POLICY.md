# BTR Vector Policy

> **Status:** Normative (BTR-3)
> **Scope:** bolt-core-sdk BTR conformance vectors

## Authority

Rust is the single authority for BTR test vectors.

| Item | Location |
|------|----------|
| Vector generator | `rust/bolt-btr/src/vectors.rs` (feature-gated: `vectors`) |
| Golden test | `rust/bolt-btr/tests/vector_golden.rs` |
| Output directory | `rust/bolt-core/test-vectors/btr/*.vectors.json` |
| TS consumer tests | `ts/bolt-core/__tests__/btr-*.test.ts` |

## Vector Categories (10)

| File | Purpose |
|------|---------|
| `btr-key-schedule.vectors.json` | Session root derivation via HKDF-SHA256 |
| `btr-transfer-ratchet.vectors.json` | Transfer root derivation |
| `btr-chain-advance.vectors.json` | Per-chunk symmetric chain KDF |
| `btr-dh-ratchet.vectors.json` | Inter-transfer DH ratchet step |
| `btr-dh-sanity.vectors.json` | X25519 cross-library sanity |
| `btr-encrypt-decrypt.vectors.json` | NaCl secretbox deterministic seal/open |
| `btr-replay-reject.vectors.json` | Replay/ordering guard |
| `btr-downgrade-negotiate.vectors.json` | Capability negotiation matrix |
| `btr-lifecycle.vectors.json` | Full lifecycle: 2 transfers × 3 chunks |
| `btr-adversarial.vectors.json` | Wrong-key decrypt + chain-index desync |

## Regeneration

```bash
cd rust && cargo test -p bolt-btr --features vectors -- --test-threads=1
```

This regenerates all 10 vector files in-place. The golden test asserts determinism (generate twice, compare).

## Review Requirement

Any change to vector output files (`rust/bolt-core/test-vectors/btr/**`) MUST:

1. Be generated by modifying `vectors.rs`, not by hand-editing JSON.
2. Pass the golden test (determinism + round-trip).
3. Pass all TS consumer tests (`npm run test` in `ts/bolt-core`).
4. Be reviewed by a human before merge — vector changes affect cross-language parity guarantees.

## CI Integration

- Rust CI (`ci-rust.yml`, `ci-gate.yml`): runs `cargo test --features vectors` which regenerates and verifies vectors.
- TS CI (`ci.yml`, `ci-gate.yml`): triggers on `rust/bolt-core/test-vectors/btr/**` changes; runs TS tests that consume vectors.
- BTR constants parity (`verify-btr-constants.sh`): runs in CI gate before TS build.
- Unified runner: `scripts/btr-conformance.sh` runs all 5 checks locally.
