# bolt-ecosystem

Governance control plane for the `the9ines/bolt-*` ecosystem, run as a **Governance OS**:
current state is derived from git (never hand-written), rules are timeless, history is
append-only, and intent lives in one small reviewed file. Hand-written "current state"
documents are retired — they go stale by nature.

## The OS

| Surface | Where | Nature |
|---------|-------|--------|
| Dashboard | `os/DASHBOARD.md` | Generated by `os/bin/status.sh`. Untracked artifact — regenerate freely, never edit or commit. |
| Intent | `os/NOW.md` | The only hand-written state file. Now / Next / Later, with a review-by date. |
| Journal | `os/log/journal.md` | Append-only history of what shipped or was decided. |
| Decisions | `os/log/decisions/` | Dated ADRs, immutable once made. |
| Rules | `CLAUDE.md` + `os/rules/` | Timeless law. No versions, counts, or status claims. |

Boot sequence: run `os/bin/status.sh`, read `os/DASHBOARD.md`, read `os/NOW.md`.
The review ritual ("give me my read"): regenerate the dashboard, compare against
NOW.md, report drift, journal anything done but unlogged.

## Routing

`os/rules/doc-routing.md` is the authoritative map of where every kind of document
lives. Stubs in other repos are pointers, never authoritative.

## Frozen history

`docs/` holds the pre-OS governance monoliths, banner-frozen 2026-07-03 — consult
freely, update never (see `docs/README.md`). Exceptions still active in that folder:
`AUDIT_TRACKER.md` (append-only audit-finding registry) and the immutable
`AUDITS/` + `evidence/` records.

## Repository landscape

Child repos are **not** included in this git repository; they are independent repos
living alongside this one on disk. Boundaries: `ARCHITECTURE.md`. Their current
state: `os/DASHBOARD.md`.
