# Document Routing — Governance OS

> **Status:** Normative. Supersedes `docs/DOC_ROUTING.md` (frozen 2026-07-03).

Every kind of fact has exactly one home. Before creating, updating, or moving any
documentation, consult this table. Do not infer or guess a target file.

## Living Surfaces (ecosystem root repo)

| Kind | Home | Rule |
|------|------|------|
| Current state (all repos) | `os/DASHBOARD.md` | Generated by `os/bin/status.sh` only. Untracked artifact. Never hand-edited, never committed. |
| Intent | `os/NOW.md` | Only hand-written state file. Small, review-dated. Finished items get a journal line and are deleted here. |
| History (ecosystem) | `os/log/journal.md` | Append-only, dated, newest first. Never edited. |
| New decisions (ADRs) | `os/log/decisions/YYYY-MM-DD-slug.md` | Dated, immutable once made. Superseding decisions get new files. |
| Rules (agent-behavioral) | root `CLAUDE.md` | SRE policy, commit/tag discipline, No-Push, doc homes. |
| Rules (ecosystem law) | `os/rules/` | Security model, validation protocol, phase discipline, this file, BTR vector policy, runbooks. |
| Audit finding registry | `docs/AUDIT_TRACKER.md` | Append-only registry of finding IDs. Status columns are historical — verify against `docs/evidence/`. |

## Per-Repo Docs

| Kind | Home | Rule |
|------|------|------|
| Release history | `<repo>/docs/CHANGELOG.md` | Append-only, still active in: bolt-core-sdk, bolt-daemon, bolt-rendezvous, localbolt, localbolt-app, localbolt-v3. |
| Repo state | `<repo>/docs/STATE.md` | **Retired 2026-07-03** — stub only. State is generated at the ecosystem root. |
| Repo contracts/specs | `<repo>/docs/*.md` | Repo-owned (e.g. daemon IPC_CONTRACT.md, sdk BOUNDARY_CONTRACT.md). Versioned with the code they describe. |

## Cross-Repo Specification Authority (unchanged)

| File | Canonical Repo | Purpose |
|------|----------------|---------|
| PROTOCOL.md | bolt-protocol | Core Bolt protocol spec |
| LOCALBOLT_PROFILE.md | bolt-protocol | LocalBolt Profile spec |
| docs/CONFORMANCE.md | bolt-protocol | Spec-to-implementation conformance matrix (CI-guarded) |

Protocol and profile semantics live exclusively in bolt-protocol. Implementation
repos must not redefine them. Stubs elsewhere (e.g. bolt-core-sdk/PROTOCOL.md) are
pointers — never update a stub.

## Read-Only Subtree Paths (unchanged)

`localbolt/signal/**` and `localbolt-app/signal/**` are vendored from bolt-rendezvous.
Never modify them in the product repos; changes flow upstream first, then
`git subtree pull --prefix=signal <remote> main --squash`.

## Frozen (never update, consult freely)

Everything in `docs/` at the ecosystem root except `AUDIT_TRACKER.md` (append-only
registry), `AUDITS/` and `evidence/` (immutable records): STATE, ROADMAP, CHANGELOG,
GOVERNANCE_WORKSTREAMS, FORWARD_BACKLOG, DOC_ROUTING, SECURITY_MODEL (extracted to
`os/rules/security-model.md`), PROTOCOL_COMPLIANCE, PROTOCOL_ENFORCEMENT,
RUST_CORE_PLAN, and the decision memos (ADR-001, BTR5) which are dated records.
